Skills

Optional packages that add scripts and workflows the assistant can run.

This page is updated automatically when the product changes.

Skills bundle scripts and instructions for specific tasks — email, browser automation, GitHub, memory search, and more. Enable skills in Mission Control or your workspace config; the assistant loads them with load_skill when relevant.

SkillWhat it does
browser-harness Thin CDP harness with extendable helpers.py for open-ended browser control.
canvas Cursor Canvas and rich analytical layouts via OpenUI compose helpers .
code_graph_rag CGR export reader + allowlisted cgr CLI .
computer-use Drive a computer via trycua/cua — host cua-driver MCP passthrough plus sandbox providers (docker/cloud/lume) through the cua CLI; opt-in; macOS-only
conventional_commit Draft git commit messages using Conventional Commits 1.0.0. Use when the operator asks to commit, record changes in git, or before running git commit after editing code (including the sevn.bot checkout).
cua-agent Autonomous GUI loop via cua-agent — model drives the screen toward a goal; requires computer-use enabled and explicit per-run operator approval (HITL).
cursor_cloud Delegate code+PR work to Cursor Cloud Agent; returns PR, dashboard, and artifact links.
defuddle Extract clean markdown content from web pages using Defuddle CLI, removing clutter and navigation to save tokens. Use instead of WebFetch when the user provides a URL to read or analyze, for online documentation, articles, blog posts, or any standard web page. Do NOT use for URLs ending in .md — those are already markdown, use WebFetch directly.
discogs-collection Discogs user collection — folders, items, value, and collection search. Writes require --confirm unless confirm_writes is disabled.
discogs-database Discogs public catalog — search artists/releases/masters/labels and read release price suggestions and marketplace stats. Works without auth (rate-limited).
discogs-identity Discogs authenticated identity — whoami smoke-test, user profiles, public lists, and release contributions.
discogs-marketplace Discogs marketplace — inventory search, listings CRUD, orders, messages, and fee lookup. Writes require --confirm unless confirm_writes is disabled.
discogs-shared Internal shared runtime for bundled Discogs skill scripts (_discogs_runtime.py). Not a model-facing research skill.
discogs-wantlist Discogs user wantlist — browse, search, and manage wants. Writes require --confirm unless confirm_writes is disabled.
email-management Search and send email through configured accounts.
gh-issues GitHub issue lifecycle — templated create via authenticated gh CLI (proxy fallback), authenticated view/watch/track with cron notify on changes, plus list/comment via integration_call.
gh-pr Pull request lifecycle — list, view, create, merge, close, reviewers via integration_call.
github-manager Work with repositories, branches, and CI from chat.
google-workspace Gmail, Calendar, Drive, and Contacts via OAuth2, with planned Sheets/Docs expansion (see google-workspace.html).
graphify Knowledge-graph orientation for code .
job-ops Discover jobs across global + Europe boards, AI fit-score them against your resume, and optionally tailor a CV summary.
json-canvas Create and edit JSON Canvas files (.canvas) with nodes, edges, groups, and connections. Use when working with .canvas files, creating visual canvases, mind maps, flowcharts, or when the user mentions Canvas files in Obsidian.
kokoro-tts Local Kokoro ONNX text-to-speech engine backing the voice TTS pipeline (kokoro backend). Not a model-facing research skill.
last30days Multi-source social research for the last 30 days (Reddit, X, YouTube, HN, Polymarket).
lcm Search and recall conversation history without losing detail.
lume Apple-Silicon VM lifecycle via the lume CLI (run/stop/ls/pull); opt-in; also a computer-use sandbox target via `cua do switch lume`
media_generation MiniMax media generation — image, video (t2v/i2v/s2v/fl2v/templates), voice clone/TTS, music — via media_generator L2 specialist with lean prompt templates and trace metadata.
mycode Scan and orient within a codebase checkout.
obsidian-bases Create and edit Obsidian Bases (.base files) with views, filters, formulas, and summaries. Use when working with .base files, creating database-like views of notes, or when the user mentions Bases, table views, card views, filters, or formulas in Obsidian.
obsidian-cli Interact with Obsidian vaults using the Obsidian CLI to read, create, search, and manage notes, tasks, properties, and more. Also supports plugin and theme development with commands to reload plugins, run JavaScript, capture errors, take screenshots, and inspect the DOM. Use when the user asks to interact with their Obsidian vault, manage notes, search vault content, perform vault operations from the command line, or develop and debug Obsidian plugins and themes.
obsidian-markdown Create and edit Obsidian Flavored Markdown with wikilinks, embeds, callouts, properties, and other Obsidian-specific syntax. Use when working with .md files in Obsidian, or when the user mentions wikilinks, callouts, frontmatter, tags, embeds, or Obsidian notes.
openwiki LLM-generated agent wiki for a codebase (LangChain OpenWiki CLI).
pdf Extract text and metadata from PDF documents.
printing-press-library Starter-pack Printing Press CLIs — ESPN, flights, movies, recipes (Go binaries on PATH).
proton-management Proton suite CLI (Python port) — full deferred feature set.
roam_code Lightweight roam-code path Q&A .
scheduling Add and manage scheduled reminders and cron-style jobs.
second_brain Ingest notes and files into your knowledge base.
sessions_management Gateway sessions, history, send, spawn, yield, and status .
sevn-diagnostics sevn.bot operator repair playbooks for `sevn doctor --with-agent`: gateway token, secrets store unlock, proxy health, model auth, browser/CDP, and voice backends. Uses the bundled solutions catalog — do not duplicate remediation text here.
skill_management Authoring workflows for generated skills; pairs with native skill_create and promote_generated_skill .
social_media_manager Browser-first social monitoring across six platforms via CDP browser; TwexAPI optional on X only.
telegram Telegram inline custom buttons and forum supergroup helpers (Bot API + allowlist/userbot hooks).
yt-dlp Download video/audio and metadata with yt-dlp (YouTube, Vimeo, X, TikTok, and allowlisted hosts).
