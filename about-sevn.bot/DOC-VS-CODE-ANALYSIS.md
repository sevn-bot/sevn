# Documentation vs code analysis — sevn.bot

> Generated by file-by-file comparison of README, `docs/readmes/`, CLI guides,
> `about-sevn.bot/` specs/PRDs/Architecture/Glossary against `src/sevn/` and repo
> surfaces. Findings are verified against paths/symbols in this tree.
>
> Branch base reviewed: `pre-0.0.1` (package `0.0.1`).

---

## 1. Executive summary

**Verdict.** Operator-facing entry docs (`README.md`, install/quick-start CLI)
are largely accurate. Generated subsystem READMEs are strong as **module
inventories** but weak as **product documentation**: titles promise behaviors
that bodies never explain. The **spec tree is largely hollow** — almost every
spec is marked `done` while the body is an “Offline scaffold …” placeholder;
code still cites missing section numbers. A handful of **high-severity factual
errors** mislead operators (secrets boundary, wrong API names, wrong config
keys, wrong normative-spec pointers).

### Severity rollup (verified)

| Severity | Count (approx.) | Theme |
|----------|-----------------|-------|
| Critical | 3 | Spec corpus unusable as contract; secrets “never in gateway” false; integrations → wrong spec |
| High | ~20 | Wrong APIs/config keys; hollow titles; STANDARD/schema drift; CI claim; tracing Logfire/OTel path |
| Medium | ~25 | Coverage gaps; package mis-attribution; CLI guide “M2” staleness; skill `>-` corruption |
| Low | ~15 | Path wording, optional extras, scope of SECURITY.md |

### Top 15 highest-impact issues

1. **Almost all specs are Offline scaffolds with `status: done`** — normative design text is gone; code still references missing §§.
2. **Mega-specs `00` / `01` / `25` (~12k lines)** — whole-tree `interfaces:` dumps + empty scaffold bodies.
3. **`docs/readmes/secrets.md` (and manifest summary): “keys never in the gateway process”** — false; gateway + Mission Control resolve/reveal secrets.
4. **`docs/readmes/integrations.md` + manifest cite `34-plugin-hooks`** — should be `29-cursor-cloud-agent`; purpose text is for plugins, not integrations.
5. **`gateway.md`: `ChannelRouter.route_inbound`** — symbol is `route_incoming`.
6. **`security.md`: `security.scanner.enabled` + `security.llmignore.retention`** — no scanner `enabled`; retention key is `retention_days`.
7. **`tracing.md` / Index: Logfire/OTel as TraceSink peers** — factory returns `None` for those kinds; export is via OTel bridge/`configure_gateway_otel`, not `OTelExporterSink` in the hot path.
8. **`storage.md` titles ActiveRunSnapshot persistence** — schema/migrations in storage; write/resume APIs in `agent/harness/snapshots.py`.
9. **`self-improve.md`: dashboard abort as live kill surface** — `abort_improve_job` exists in facade; no dashboard/Telegram/CLI wiring found.
10. **`tools.md` framed as `@sevn_tool` plugin inventory** — table is module files, not tool names; only reserved entry-point plugin exists.
11. **`CONTRIBUTING.md`: “what CI runs” = `make ci`** — PR CI equals `make ci` **minus** `security`.
12. **`STANDARD.md` §F exports / §C0 guide schema** — package exports `render_profile` / `PROFILE_TEMPLATES`, not `render`/`check`/`generate_index`; live `profile_schemas.py` differs from STANDARD YAML.
13. **`spec-16` sources = `src/sevn/runtime/**`** — harness lives in `agent/harness/` (+ gateway harness config); summary corrupted as `45# Harness…`.
14. **`spec-06` sources = `src/sevn/secrets/**`** — trust boundary backends are `src/sevn/security/secrets/`.
15. **`ARCHITECTURE.md` → missing `specs-index.md`; `GLOSSARY.md` has empty `## Terms`.**

## 3. Cross-cutting themes

### 3.1 Generated READMEs are inventories, not guides

Most non-curated (and many curated) subsystem READMEs are regenerated module
tables. Manifest summaries promise product semantics (routes, provider chains,
wiki layouts, webhook/cron flows, SPA tabs). Bodies rarely deliver those
semantics. Operators reading titles get a false sense of coverage.

### 3.2 Spec status is unreliable

Almost every file under `about-sevn.bot/specs/` contains seven “Offline scaffold …”
sections. Index still lists most as `done`. Only `36-sub-agents` is substantially
authored; `27-second-brain` is partial. Code comments cite § numbers that no
longer exist in the files.

### 3.3 Trust-boundary wording is absolute and wrong

Multiple docs claim credentials “never” enter the gateway. Reality: provider API
keys are intended for the egress proxy; **channel tokens, gateway token, and
Mission Control secret reveal** use the gateway process and
`secrets_chain_from_workspace`.

### 3.4 Manifest / INDEX “ok” staleness is not semantic correctness

`INDEX.md` status `ok` means fingerprint/structure gates, not “claims match
behavior.” Several `ok` rows carry false headlines (secrets, integrations).

### 3.5 Packages without README coverage (by curated design)

Present in code, absent from README catalog: `evolution/`, `plugins/`,
`browser/`, `pdf/`, `branding/`, `deploy/`, `provisioning/`, `coding_agents/`,
`media/`, `infrastructure/`, `standards/`, `prompts/`. STANDARD intentionally
avoids one-README-per-package — but evolution/plugins/browser are load-bearing
and have no subsystem README at all.
