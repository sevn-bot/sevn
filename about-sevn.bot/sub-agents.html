Sub-agents

Two-level concurrent agents — tracked role runs (L1) that may spawn workers and specialists (L2), with limits, kill controls, and multi queue mode.

sevn.bot runs up to N concurrent level-1 sub-agents per role (triager, tier B, C, D — default 5 each). Each level-1 run may spawn up to M level-2 workers (default 3 per parent). Level 2 cannot spawn further — the depth cap is hard at two levels.

Level 1 — tracked role runs

Every triager and tier B/C/D executor turn registers as a level-1 sub-agent run in the gateway registry. Each run gets a short id (e.g. a1f3), is traced, counts against per-role limits, and is killable from Mission Control, Telegram /config → Sub-agents, or sevn subagents kill.

Level 2 — workers and specialists

Tier B (and C/D where allowed) exposes the spawn_subagent tool. By default it is fire-and-forget: the tool returns a run id immediately and announces the result back when the worker finishes. Pass wait: true when a skill needs the artifact inline (bounded by the parent turn's cascade budget).

Named specialists (first example: media_generator on MiniMax-3) are configured under subagents.specialists. The triager may attach specialist_grants to a tier-B dispatch; tier B may also spawn assigned specialists directly.

Multi queue mode

When gateway.queue_mode (or per-channel busy_input_mode) is multi, a message arriving while a session is busy is classified as related steer, supersede / cancel, or new task. A new task spawns a fresh level-1 tier-B sub-agent on the same session. Classifier timeout or limit exceeded falls back to steer with a one-line operator notice.

Operator surfaces

  • Mission Control — Sub-agents panel: L1/L2 count chips, running table, kill buttons, recent history
  • Telegram settings — Sub-agents section: limits, live counts, Running kill submenu
  • sevn subagents list|kill|limits and sevn config subagents on the CLI
  • Settings — subagents.* and gateway.queue_mode keys
