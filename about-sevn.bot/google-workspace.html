Google Workspace

Gmail, Calendar, Drive, Sheets, Docs, and Contacts via OAuth2 — planned bundled skill with Hermes parity, adapted for sevn.

The google-workspace skill brings full Google Workspace API access to sevn.bot, modeled on the Hermes Agent google-workspace skill. It uses OAuth2 with automatic token refresh and prefers the official Google Workspace CLI (gws) when installed.

Status

This skill is planned (spec 38). Today, sevn covers partial Google access through:

  • browser — Gmail read/search (and gated compose/reply), Google Search, Maps, YouTube
  • email-management — multi-account IMAP/SMTP (Gmail app passwords; no Cloud project)

When shipped, google-workspace becomes the primary path for Calendar, Drive, Sheets, Docs, and OAuth Gmail API writes.

How the agent uses the skill

  1. Triager selects google-workspace for relevant intents.
  2. Agent calls load_skill("google-workspace") for setup prose and script capabilities.
  3. Agent runs operations with run_skill_script, passing argv to scripts/google_api.py or scripts/setup.py.

Example (planned):

run_skill_script(
  "google-workspace",
  "scripts/google_api.py",
  ["gmail", "search", "is:unread", "--max", "10"]
)

First-time OAuth setup (planned flow)

  1. Operator creates a Google Cloud project and enables Gmail, Calendar, Drive, Sheets, Docs, and People APIs.
  2. Download Desktop OAuth client JSON; agent runs scripts/setup.py --client-secret PATH.
  3. Agent returns auth URL; operator approves in browser and pastes redirect URL.
  4. scripts/setup.py --auth-code … stores token under .sevn/google_token.json (auto-refresh).

Credentials never appear in chat or script JSON output.

Safety rules

  • Confirm with the operator before sending email, creating/deleting calendar events, uploading or sharing Drive files, or modifying Sheets/Docs.
  • Prefer Drive trash over permanent delete.
  • Calendar times must use ISO 8601 with timezone offset or UTC (Z).
