03 · my sevn.bot

My Sevn.bot

Bot name, repo URL, daily sync, GitHub auth, workspace backup, and self-improve defaults.

How your assistant introduces itself and which upstream sevn.bot repository powers evolution workflows.

Keeping https://github.com/sevn-bot/sevn means core checkout updates can overwrite local core changes. Evolution and self-improve are limited to non-core tools and skills. The upside is automatic bug and feature issue filing against upstream.

GitHub authentication

OAuth or a personal access token. Stored encrypted as integration.github.token — never written to sevn.json.

Register callback http://127.0.0.1:8844/api/github/oauth/callback on your OAuth app. Credentials stay in this wizard session only.

Not connected

Create a token with repo and read:user scopes (GitHub token settings). Optional workflow for Actions-aware evolution.

Workspace backup

Private GitHub repo for workspace snapshots. Default name: {your_login}.mysevnbackup.

Self-improve

What is self-improve?

Trajectory sampling and bounded patch proposals from real conversations. When enabled, sevn.bot can propose small improvements to tools, skills, and workspace memory under operator control.

GitHub hub vs local-only

GitHub hub mirrors issues and pull requests to your configured repo — evolution pipelines, bug filing, and spec-kit flows can open and track work on GitHub.

Local-only keeps self-improve proposals on disk with no remote issue mirror. Uncheck this box when you do not want GitHub API calls for the hub.

07 · secrets

Secrets backend

Where API keys and tokens live outside sevn.json.

Provider keys, bot tokens, and your Telegram api_hash never belong in sevn.json. Pick where sevn stores them so the gateway and CLI can read them back at runtime. All values you've typed in the wizard get written to whichever backend you choose when you Save & promote.

Encrypted file

An encrypted keystore already exists for this install. Enter your passphrase below, or replace the keystore and re-enter credentials on the Channels step.

A single AES-GCM-256 file under your workspace. Safe by default — but anyone who steals the file and your passphrase can read your secrets. Pick a passphrase you don't reuse.

Default: <workspace>/.sevn/secrets/store.enc

Stored in the chain itself so the same gateway process can decrypt later. For daemon mode, export SEVN_SECRETS_PASSPHRASE in your launchd/systemd unit so sevn can decrypt at boot — the wizard will print a reminder on Save. This sets key_source: passphrase (the default). Headless/CI installs can instead set secrets_backend.encrypted_file.key_source: master_key in sevn.json and supply a 64-hex SEVN_SECRETS_MASTER_KEY.

Passphrases do not match.

OpenBao

OpenBao is the OSS HashiCorp Vault fork. The wizard captures the backend type only; finish wiring the address, mount, and token by editing sevn.json by hand once setup is complete — see openbao.org/docs.

08 · sandbox

Sandbox

How tool runs are isolated from your host.

Sevn runs tool code (shell commands, Python snippets, fetches) inside a sandbox so a buggy or hostile tool cannot touch your real filesystem, secrets, or network. Pick the mode that matches your machine — Use preset default is the right answer unless you have a specific reason to override.

Inherits the sandbox mode from the profile you picked. The Good-value preset chooses pyodide_deno on macOS hosts and docker on Linux/Docker hosts — both safe defaults you don't need to think about.

Runs each tool call in a throwaway Docker container — the strongest isolation. Best when you already have Docker Desktop (macOS) or a docker daemon (Linux) running. Adds ~1–3 s per tool call for container boot, but keeps host filesystem and network unreachable from tool code.

Pick this if: you want production-grade safety, or your tools need binaries that aren't available in the WASM sandbox.

Runs Python via Pyodide (WASM) and JavaScript via Deno's permissions model. No Docker needed; near-instant startup; safe by construction.

Pick this if: you're on a laptop without Docker, or you want the fastest possible tool turnaround. Note: tools that need native binaries (e.g. ffmpeg, custom system commands) won't work here.

09 · tunnel

Public access

Cloudflare, ngrok, or Tailscale — or polling only.

Telegram long-polling works without a tunnel. Pick a tunnel only if you need HTTPS webhooks reachable from the public internet. Not sure? Leave None selected.

Your bot pulls updates from Telegram every few seconds. No inbound ports, no extra software. Works on any laptop or home server.

HTTPS reachable only from devices on your Tailscale account. Best if you already use Tailscale.

  1. Install Tailscale and sign in on this machine.
  2. Run tailscale serve --bg 3001 in a terminal once setup completes.
  3. Optional: enter the tailnet hostname below so sevn can announce it.

Exposes the gateway to the public internet through Tailscale's edge. Requires Funnel to be enabled on your tailnet.

  1. Install Tailscale and enable Funnel in the admin console ACLs.
  2. Run tailscale funnel --bg 3001 after onboarding.
  3. A gateway auth token is auto-generated on the workspace step and stored in your secrets chain — required when the gateway is reachable from the internet.

Cloudflare runs a tiny daemon (cloudflared) that punches an outbound connection to their edge — no firewall changes needed. Free for personal use.

  1. Sign in to Cloudflare Zero Trust.
  2. Go to Networks → Tunnels, click Create a tunnel, pick Cloudflared, give it a name.
  3. Cloudflare shows a one-line cloudflared service install <TOKEN> command. Copy just the TOKEN (the long string at the end) and paste it below.
  4. Alternatively, if you already ran cloudflared tunnel login, point sevn at the JSON credentials file (usually ~/.cloudflared/<tunnel-uuid>.json).

Easiest public HTTPS. The free tier rotates URLs on each restart; demos only, not production.

  1. Sign up at ngrok.com.
  2. Copy your authtoken from the authtoken page and paste it below.
  3. After saving, sevn will run ngrok http 3001 in the background when the gateway starts.
