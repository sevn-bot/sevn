## Memory subsystems

Sevn layers memory in several places; this file is the durable scratchpad. For how each layer is used in practice, see **`AGENTS.md`** (Memory subsystems):

- **Daily logs** — `memory/YYYY-MM-DD.md` for session-local notes
- **LCM** — cross-session search over indexed memory
- **Honcho** — user profile flush into `USER.md` when enabled
- **Dreaming** — background consolidation when enabled

Keep this file focused on facts you want promoted or recalled often; do not duplicate the full subsystem guide here.

## How to use this file

- Add **durable facts** you want remembered across sessions (people, projects, decisions, constraints).
- Prefer dated bullets when facts may go stale.
- The agent may append via pre-compaction flush or Dreaming when those features are enabled — review promotions periodically.
