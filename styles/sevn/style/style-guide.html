The design system for sevn.bot

One technical user. One bot. A confident, calm, slightly playful copilot — never corporate, never childish, never twee. This document is the locked compass; everything in /style/ compiles against the tokens below.

Code blocks

Syntax palette is muted; primary marks keywords, accent flags builtins-of-note, warning-amber strings, primary-light numbers.

routing.py python 
from sevn import triage, run
from sevn.policy import Policy

def handle(msg: str) -> Run:
    # tier C runs survive boot — see /policy/tiers.md
    tier = triage(msg, policy=Policy.current())
    return run(msg, tier=tier, budget=0.05)

Logo lockups

The mark is an ASCII-style “7” in signal red and clear-sky blue — two colors only, no gradient. The wordmark pairs sevn in heavy Inter Tight with .bot in JetBrains Mono — a deliberate domain-style suffix.

logo-mark.svgcompact · 80×80
logo-primary.svgon base · 050

logo-all-white.svgon brand gradient
avatar-github.svg280×280
Logo usage
  • ✓ Maintain clear-space equal to the height of the mark.
  • ✓ Use the all-white version on photographic backgrounds.
  • ✓ Minimum size: 22px for logo-mark, 96px wide for the full lockup.
  • ✓ The brand-glow shadow is the only legal drop shadow.
  • ✕ Don’t swap the bar/diagonal colors — red stays the bar, blue stays the diagonal.
  • ✕ Don’t apply gradients, glows, or fills to the mark.
  • ✕ Don’t stretch, skew, or outline the wordmark.
  • ✕ Don’t typeset “sevn.bot” in any font other than Inter Tight + JetBrains Mono.
